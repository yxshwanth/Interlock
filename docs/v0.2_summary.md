## Phase 1 ‚Äî HTTP/SSE Transport Interception

### Done when

> Full trifecta demo runs against an HTTP MCP server, not just STDIO. ‚Äî **Met**

### Shipped

- Streamable HTTP ([MCP 2025-11-25](https://modelcontextprotocol.io/specification/2025-11-25/basic/transports/streamable-http)): `POST /mcp`, `Mcp-Session-Id`, JSON + SSE
- `internal/proxy/dispatch.go` ‚Äî transport-agnostic dispatch shared with STDIO
- `internal/proxy/http/` ‚Äî transport, headers, response, client
- **Inspect-then-forward** ‚Äî full JSON-RPC body before policy; SSE only after complete backend response; blocked calls return JSON immediately
- Config: `transport.mode`, `listen`, `endpoint`, `protocol_version`, `prefer_sse_responses`
- Demos: `make demo-http`, `make demo-http-ebpf`, quiet variants
- Example configs: `interlock-http.yaml`, `interlock-http-monitor.yaml`

### Scope decisions

- Agent-facing HTTP only; **backends stay STDIO** (eBPF PID dependency)
- Bind `127.0.0.1` ‚Äî inside trust boundary; TLS/MITM deferred
- STDIO unchanged and still default

### Deferred (not v0.2)

- HTTP upstream MCP backends
- GET `/mcp` listen streams, 2026-07-28 stateless protocol

## Phase 2 ‚Äî Multi-Session Concurrency and Attribution

### Done when

> Two poisoned sessions run concurrently, each correctly attributed, neither leaking state. ‚Äî **Met**

### Shipped

- `SessionManager` + `SessionRuntime` ‚Äî per-session backend pools spawned on HTTP `initialize`
- `PIDRegistry` ‚Äî `(pid, /proc start_time)` ‚Üí `{session_id, server_id}`
- eBPF dynamic `AddPID` / `RemovePID` on session spawn and cleanup
- Removed `FirstSessionID`; `IngestSyscall` requires explicit `SessionID`
- Unattributed eBPF events: `[SECURITY]` log + `SecurityAuditEvent` in `events.jsonl` (#10)
- Config: `sessions.max_concurrent` (32), `sessions.idle_timeout` (30m)
- Tests: `TestConcurrentDualSession_VariantA_Block`, PID reuse, session isolation
- Demo: `make demo-http-concurrent`

### Review hardening (#10)

- CI **`race` job** ‚Äî `go test -race` on `./internal/proxy/...` and `./internal/engine/...`
- Concurrent overlap barriers in dual-session and `SessionManager` tests
- DoS limitation documented (per-session pool bound)
- `TestSessionStore_ConcurrentAccess` rewritten to exercise `Engine` under `-race`

### Honest limitations

- Per-session pools = bounded DoS surface (`max_concurrent √ó servers`)
- Just-died PID window: audit-logged, not tripped
- Shared `PIDRegistry` / `SessionManager` ‚Äî race CI is the safety net

## Architecture (v0.2 end state)

```
Agent(s) ‚îÄ‚îÄ HTTP POST /mcp ‚îÄ‚îÄ‚ñ∫ SessionManager ‚îÄ‚îÄ‚ñ∫ SessionRuntime (per session)
                                      ‚îÇ                    ‚îÇ
                                      ‚îÇ                    ‚îú‚îÄ‚îÄ STDIO backends (tickets/messenger/exfil)
                                      ‚îÇ                    ‚îî‚îÄ‚îÄ PIDRegistry ‚îÄ‚îÄ‚ñ∫ eBPF pid_filter
                                      ‚ñº
                                 engine (per session_id)
                                   ‚îú‚îÄ‚îÄ trifecta state machine
                                   ‚îú‚îÄ‚îÄ CheckOverlap (5 forms √ó N tainted)
                                   ‚îî‚îÄ‚îÄ EvidenceSink (JSONL default | SQLite opt-in)
                                      ‚ñ≤
                                      ‚îÇ SyscallEvent + SessionID
                                 eBPF sensor (connect + drop_count)
```

Two planes, one engine ‚Äî session-aware on both paths, encoding-aware on Variant A.
